    Troubleshooting Guide

    Common issues and their solutions when working with LegoCity.

    Quick Diagnostics

    Run these checks first:

    # Check all services
pnpm run check:all

# Or check individually:
node --version        # Should be 18+
pnpm --version        # Should be 8+
mongosh --eval "db.version()"  # MongoDB running?
curl http://localhost:3000/api/health  # App running?

    Installation Issues

    Node.js Version Mismatch

    Problem: error:0308010C:digital envelope routines::unsupported

    Cause: Node.js version incompatibility

    Solution:

    # Check required version
cat .nvmrc

# Install correct version
nvm install 18
nvm use 18

# Verify
node --version

    pnpm Not Found

    Problem: pnpm: command not found

    Solution:

    # Install pnpm globally
npm install -g pnpm

# Or use Corepack (Node.js 16+)
corepack enable
corepack prepare pnpm@latest --activate

    MongoDB Connection Failed

    Problem: MongoServerError: connect ECONNREFUSED 127.0.0.1:27017

    Cause: MongoDB not running or wrong connection string

    Solutions:

    ```powershell # Start MongoDB service net start MongoDB

    # Check status
sc query MongoDB

# Or start manually
mongod --dbpath C:\data\db
```

    ```bash # Start with Homebrew brew services start mongodb-community

    # Or manually
mongod --config /usr/local/etc/mongod.conf
```

    ```bash # Start service sudo systemctl start mongod

    # Enable on boot
sudo systemctl enable mongod

# Check status
sudo systemctl status mongod
```

    ```bash # Start MongoDB container docker run -d -p 27017:27017 --name mongodb mongo:6

    # Or with Docker Compose
docker compose up -d mongodb
```

    Port Already in Use

    Problem: Error: listen EADDRINUSE: address already in use :::3000

    Solutions:

    ```powershell # Find process using port netstat -ano | findstr :3000

    # Kill process (replace PID)
taskkill /PID [PID] /F

# Or use npx
npx kill-port 3000
```

    ```bash # Find and kill process lsof -ti:3000 | xargs kill -9

    # Or use npx
npx kill-port 3000
```

    Alternative: Use different port

    PORT=3001 pnpm dev

    Development Issues

    Hot Reload Not Working

    Problem: Changes not reflecting in browser

    Solutions:

    1. Clear Next.js cache:
    rm -rf .next
pnpm dev
    1. Check file watcher limits (Linux):
    # Increase limit
echo fs.inotify.max_user_watches=524288 | sudo tee -a /etc/sysctl.conf
sudo sysctl -p
    1. Restart dev server: 
      # Kill server (Ctrl+C)
pnpm dev

    TypeScript Errors After Update

    Problem: Type errors after updating dependencies

    Solution:

    # Regenerate Payload types
pnpm payload generate:types

# Clear TypeScript cache
rm -rf node_modules/.cache

# Reinstall dependencies
rm -rf node_modules
pnpm install

    Build Fails

    Problem: Build failed with errors

    Common causes and solutions:

    1. Type errors:
    pnpm type-check
# Fix reported errors
    1. Lint errors:
    pnpm lint:fix
    1. Out of memory:
    # Increase Node memory
NODE_OPTIONS="--max-old-space-size=4096" pnpm build
    1. Module not found: 
      # Reinstall dependencies
rm -rf node_modules pnpm-lock.yaml
pnpm install

    Runtime Issues

    White Screen / Blank Page

    Problem: Page loads but shows nothing

    Debugging steps:

    1. Check browser console for JavaScript errors

    2. Check Network tab for failed requests

    3. Verify environment variables:

    # Required variables set?
echo $DATABASE_URI
echo $PAYLOAD_SECRET
echo $NEXT_PUBLIC_SERVER_URL
    1. Check server logs: 
      pnpm dev
# Look for errors in terminal

    Map Not Rendering

    Problem: Map area is blank or shows error

    Solutions:

    1. Verify Mapbox token:
    # Check token is set
echo $NEXT_PUBLIC_MAPBOX_TOKEN

# Test token
curl "https://api.mapbox.com/v4/mapbox.mapbox-streets-v8.json?access_token=$NEXT_PUBLIC_MAPBOX_TOKEN"

    1. Check console for errors:

    2. Open browser DevTools → Console

    3. Look for Mapbox errors

    4. Verify token scope:

    5. Token must have styles:read and fonts:read scopes
    6. Create new token at mapbox.com

    Admin Panel 404

    Problem: /admin route not found

    Solutions:

    1. Rebuild application:
    rm -rf .next
pnpm build
pnpm start
    1. Check Payload config:
    // payload.config.ts
admin: {
  autoLogin: false,  // Should be false in production
}
    1. Verify admin routes exist: 
      ls src/app/\(payload\)/admin

    API Route Returns 500

    Problem: API endpoint throws server error

    Debugging:

    1. Check server logs for error details

    2. Test with curl:

    curl -v http://localhost:3000/api/your-endpoint
    1. Add logging: 
      export async function GET(req: Request) {
  try {
    console.log("Request:", req.url);
    // ... your code
  } catch (error) {
    console.error("Error:", error);
    return Response.json({ error: error.message }, { status: 500 });
  }
}

    Database Issues

    Database Connection Slow

    Problem: Slow queries or timeouts

    Solutions:

    1. Add indexes:
    // In MongoDB shell
db.pages.createIndex({ slug: 1 });
db.posts.createIndex({ publishedAt: -1 });
    1. Monitor slow queries:
    db.setProfilingLevel(2);
db.system.profile.find().sort({ ts: -1 }).limit(10);
    1. Check database stats: 
      mongosh legocity --eval "db.stats()"

    Collection Not Found

    Problem: Collection 'pages' not found

    Solutions:

    1. Check collection name matches config:
    // collections/Pages.ts
slug: "pages"; // Must match
    1. Verify database:
    mongosh legocity --eval "show collections"
    1. Seed database: 
      pnpm seed

    Database Migration Failed

    Problem: Schema changes not applied

    Solution:

    # Backup first!
mongodump --db legocity --out ./backup

# Drop and recreate
mongosh legocity --eval "db.dropDatabase()"
pnpm seed

# Or restore backup
mongorestore --db legocity ./backup/legocity

    Docker Issues

    Container Exits Immediately

    Problem: Docker container starts then stops

    Debug:

    # Check logs
docker compose logs dashboard

# Check for errors in build
docker compose build --no-cache dashboard

# Run interactively
docker compose run --rm dashboard sh

    Cannot Connect to Service

    Problem: Frontend can't reach MongoDB/Orion

    Solutions:

    1. Use service names, not localhost:
    # ❌ Wrong
DATABASE_URI=mongodb://localhost:27017/legocity

# ✅ Correct
DATABASE_URI=mongodb://mongodb:27017/legocity
    1. Check network:
    docker compose ps
docker network ls
    1. Test connectivity: 
      docker compose exec dashboard ping mongodb

    Volume Permission Issues

    Problem: Permission denied errors (Linux)

    Solution:

    # Fix ownership
sudo chown -R $USER:$USER ./dashboard

# Or run as current user
docker compose run --user "$(id -u):$(id -g)" dashboard pnpm dev

    Production Issues

    502 Bad Gateway

    Problem: Nginx shows 502 error

    Solutions:

    1. Check app is running:
    pm2 status
# Or
systemctl status legocity
    1. Check Nginx config:
    sudo nginx -t
sudo systemctl reload nginx
    1. Check logs:
    # App logs
pm2 logs

# Nginx logs
sudo tail -f /var/log/nginx/error.log

    Memory Leak

    Problem: Application memory usage grows over time

    Solutions:

    1. Monitor memory:
    # PM2 monitoring
pm2 monit

# Or use htop
htop
    1. Set memory limit:
    // ecosystem.config.js
module.exports = {
  apps: [
    {
      name: "legocity",
      script: "node_modules/next/dist/bin/next",
      args: "start",
      max_memory_restart: "1G",
    },
  ],
};
    1. Profile memory: 
      node --inspect node_modules/next/dist/bin/next start
# Use Chrome DevTools to profile

    SSL Certificate Issues

    Problem: HTTPS not working or certificate errors

    Solutions:

    1. Check certificate validity:
    sudo certbot certificates
    1. Renew certificate:
    sudo certbot renew
sudo systemctl reload nginx
    1. Test SSL config: 
      curl -vI https://your-domain.com

    Performance Issues

    Slow Page Load

    Problem: Pages take too long to load

    Solutions:

    1. Enable caching:
    // app/page.tsx
export const revalidate = 60; // ISR every 60s
    1. Optimize images:
    import Image from "next/image";

<Image src="/image.jpg" width={800} height={600} alt="Description" />;
    1. Use dynamic imports:
    const HeavyComponent = dynamic(() => import("./HeavyComponent"), {
  loading: () => <p>Loading...</p>,
});
    1. Analyze bundle: 
      pnpm build --analyze

    High CPU Usage

    Problem: CPU spikes during normal operation

    Check:

    1. Profile application:
    node --prof node_modules/next/dist/bin/next start
node --prof-process isolate-*.log > profile.txt

    1. Check for infinite loops in code

    2. Monitor processes: 

      top -p $(pgrep node)

    Getting More Help

    Collect Diagnostic Info

    Before asking for help, collect:

    # System info
node --version
pnpm --version
mongosh --version

# App info
cat package.json | grep version

# Environment
echo $NODE_ENV
env | grep -E '(DATABASE|PAYLOAD|NEXT_PUBLIC)'

# Logs
pnpm dev > debug.log 2>&1

# Health check
curl http://localhost:3000/api/health
